| Version | Supported |

## Reporting a Vulnerability

If you discover a security vulnerability in JM-ADK, please report it responsibly:

1. **Do NOT** open a public GitHub issue
2. **Email**: javier@metodologia.info
3. **Subject**: `[SECURITY] JM-ADK: Brief description`
4. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### Response Timeline

| Action | Timeline |
|--------|----------|
| Acknowledgment | Within 48 hours |
| Assessment | Within 5 business days |
| Fix (critical) | Within 7 days |
| Fix (moderate) | Within 30 days |
| Public disclosure | After fix is released |

## Security Practices

### What JM-ADK Does

- **G0 Quality Gate**: Every output passes a security scan before delivery
- **No secrets in code**: The CI pipeline scans for exposed API keys, tokens, and credentials
- **Evidence tags**: Security claims are tagged `[CODE]` or `[CONFIG]` for traceability
- **Firestore Security Rules**: Generated projects always include security rules
- **CSP-ready headers**: Deployment configs include security headers by default
- **Input validation**: Generated code validates both client-side and server-side

### What JM-ADK Does NOT Do

- Store or transmit user credentials
- Access external services without explicit user configuration
- Execute code outside of Claude Code's sandboxed environment
- Ship with any hardcoded secrets or API keys

### For Users of Generated Code

Code generated by JM-ADK follows security best practices, but you are responsible for:

- Reviewing generated Firestore Security Rules before deployment
- Managing your own API keys and secrets via environment variables
- Keeping dependencies updated (`npm audit`)
- Enabling HTTPS on your deployment target
- Testing security rules with the Firebase Emulator before going live

## Scope

This security policy covers:

- The JM-ADK plugin code (agents, skills, commands, prompts, scripts)
- The project templates in `templates/`
- The automation scripts in `scripts/` and `.agent/scripts/`

This does **not** cover:

- Code generated by JM-ADK for user projects (user's responsibility after generation)
- Third-party MCP servers configured by the user
- Claude Code itself (report to [Anthropic](https://anthropic.com/security))
